Start Up

To run the application in networked client mode, type the following command on the command line:

     java -jar runme.jar

When the application starts up in this mode for the first time, you should see the following dialog:



Figure 1: The Connect to Server dialog

How to Connect to a Server

To connect to a server that is online, follow these steps:

  1. Type the server location in the Server Location text field. The server location can be a host name (e.g. java.sun.com) or an IP address (e.g. 192.168.1.100).
  2. Type the port number on the server in the Port text field. The port must be between 0 and 65,535, inclusive.
  3. After inputing the server location and port number, the Connect button will be enabled. Click on this button to connect to the server. If all goes well, you should see the dialog in Figure 3.
  4. Once connected, the text fields in the dialog will be disabled, the text in the Connect button will change to Disconnect, and the OK button will be enabled. Click on the OK button to close the Connect to Server dialog and open the main application window. (See the Main Application Window section below.)


Figure 2: Entering the server IP address and port number




Figure 3: The client is now connected to the server

How to Disconnect from a Server

To disconnect from a server and change the server location and/or port number, click on the Disconnect button. The text in the Disconnect button will revert back to Connect, and the OK button will be disabled again until the client is connected to a server.

To quit the application, click on the Cancel button. Note that your changes will not be automatically saved, so if you run the application in networked client mode again, you will have to re-enter the server location and port number.



Figure 4: The state of the components after a connection has been established

Status Icons

The following icons appear at the bottom of the Connect to Server dialog in the status bar and indicate the status of a connection between a client and a server.

Not connected to server. The client is not connected to a server.
Connecting to server. The client is trying to establish a connection to a server.
Connected to server. The client is connected to the server at the given server location and port.

Main Application Window

The main application window looks and functions the same as the main application window when running the application in standalone client mode. The only difference is that the status message at the bottom of the window in the status bar displays the status of the server connection as well as the server IP address and port number.



Figure 5: The main application window when running in networked client mode


If you close the main application window when running in networked client mode, the server location and port number will be automatically saved to a file called suncertify.properties. When the application is started in this mode again, the Connect to Server dialog will be bypassed and the main application window will be shown along with a message indicating that the client is connected to the server at the given IP address and port number, as shown in Figure 6 below.



Figure 6: The main application window and a message indicating the status of the server connection


If a client loses the connection to the server, which could happen when the server is shut down, for example, a message will appear and notify you that the server connection has been lost and thus the application will shut down.



Figure 7: A message indicating that the server connection has been lost and thus the application will shut down


If the server is offline and the application is run in networked client mode using the server location and port number that were saved to suncertify.properties, you will see the following message:



Figure 8: A message indicating that the server connection couldn't be established

Important Notes

  • The client will be disconnected from the server automatically if there is no activity within 10 minutes after the previous activity (e.g. booking a contractor). If you try to book, unbook, or search for a contractor after 10 minutes have passed, you will see the error message shown in Figure 7.
